# Princess OS 구현 감사

## 현재 구현된 핵심

- 계정별 Zustand 저장소와 Supabase RLS 기반 데이터 격리
- React Router 기반 상위·상세 URL과 브라우저 이전/다음 이동
- 메인퀘스트와 선택적으로 연결되는 일일·서브퀘스트
- 데스크톱 드래그 연결과 모바일 편집 대안
- 일정·기간 일정·RRULE 반복 표시
- 도서관 카테고리, 즐겨찾기 서가, 검색, 필터, 정렬, 페이지네이션
- 다이어리 완료 퀘스트 스냅샷
- 비망록·인연록과 리타의 확인형 저장 초안
- 문서·음성 요약과 명함 이미지 분석
- 사용자 설정, 방별 배경, 버전형 JSON 병합 가져오기/내보내기
- 앱 실행 중 일정·마감·검토 항목 인앱 알림

## 2026-07-13 안정화

- 전역 `.app-shell { zoom: 1.25 }`를 제거했다. viewport 계산을 변경하는 전역 확대는 모바일 오버플로, 모달 잘림, sticky 위치 오류를 만들 수 있으므로 사용하지 않는다.
- 반복 퀘스트의 완료 상태를 원본 `quests`에서 지우고 되살리는 방식을 폐기했다.
- `quest_completion_logs`를 추가하여 서비스일별 완료 및 완료 취소를 저장한다.
- 기존 반복 퀘스트의 `completed_at`은 마이그레이션에서 날짜별 로그로 옮긴 뒤 템플릿을 active로 정규화한다.
- 다이어리는 날짜별 완료 로그를 사용하므로 과거 반복 퀘스트 완료를 가져올 수 있다.
- 리타 함수는 누락된 `ANTHROPIC_API_KEY`와 `CLAUDE_MODEL`을 구분하며 지원하지 않는 파일 형식과 외부 제공자 실패를 올바른 HTTP 상태로 반환한다.
- 삭제됐던 ERD, 데이터 사전, 배포 문서를 현재 구조 기준으로 복원했다.

## 운영 전 남은 검증

- `202607130002_quest_completion_logs.sql`을 production Supabase에 적용한다.
- `verify_canonical_schema.sql`에서 필수 컬럼, RLS, 정책, private bucket을 확인한다.
- 두 계정과 두 기기로 반복 완료/취소/날짜 전환을 검증한다.
- 정상화된 의존성 환경에서 `pnpm lint`, `pnpm build`를 통과시킨다.
- OS 푸시, 백그라운드 알림, 반복 일정 개별 회차 수정은 별도의 서버/예외 모델 작업으로 남는다.

## 2026-07-13 사용성·성능 패치 (v0.3.0)

- 모든 공통 팝업과 전체 검색을 sticky header 아래, footer 위의 body area로 제한했다. 원본 페이지 스크롤은 잠그고 긴 폼은 팝업 내부에서만 스크롤한다.
- 퀘스트처럼 긴 팝업의 제목과 하단 작업 버튼을 내부 스크롤 중에도 유지한다.
- 루멘 비밀정원은 공통 header/footer를 유지하면서 `100dvh` 안에서 끝나는 단일 화면으로 변경했다.
- 인연록에 사용자 정의 그룹과 다중 그룹 소속을 추가했다. 그룹을 지워도 인연 데이터는 삭제되지 않고 미분류로 남는다.
- 리타의 명함·인연록 초안 확인 단계에서 보관할 그룹을 선택할 수 있다.
- `relationship_groups`, `relationship_group_members`와 계정 소유권 RLS를 추가했다. 운영 DB에는 `202607130003_relationship_groups.sql` 적용이 필요하다.
- 방별 배경의 서명 URL은 계정 단위로 캐시하고, 페이지 이동 때마다 재조회하지 않는다.
- 상위 페이지를 route chunk로 분리하고 책 표지용 270px WebP 썸네일을 추가해 도서관 초기 이미지 전송량을 약 6.2MB에서 약 205KB로 줄였다.
- 모바일에서는 본문 Glassmorphism blur를 끄고 데스크톱에서도 blur 강도를 낮춰 스크롤 합성 비용을 줄였다.
- `pnpm lint`와 `pnpm build`를 통과했으며, 데스크톱 브라우저에서 팝업 경계·내부 스크롤·정원 무스크롤·인연록 그룹 UI를 확인했다.
